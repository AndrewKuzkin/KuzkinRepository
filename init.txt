1. Привязать свою  директорию к репозиторию
git init [<твоя директорияy>]
2. Добавить удаленный репозиторий, к которому вы хотите привязать свою локальную директорию, с помощью команды:‍
git remote add origin <URL_удаленного_репозитория>
Здесь <URL_удаленного_репозитория> - это URL вашего удаленного репозитория. Например:‍
git remote add origin https://github.com/username/repository.git
3. Добавить файлы в индекс и сделать коммит, чтобы зафиксировать текущее состояние вашей директории:‍
git add .
   git commit -m "Initial commit"
   ‍
4. Отправить изменения на удаленный репозиторий с помощью команды:‍
git push -u origin master
Эта команда отправит ваш локальный коммит на ветку master удаленного репозитория.‍
‍Теперь ваша локальная директория привязана к удаленному репозиторию. Вы можете вносить изменения в свои файлы, делать коммиты и отправлять их на удаленный репозиторий с помощью команд git add, git commit и git push.‍
---
Для создания репозитория на GitHub и его привязки к локальной папке, следуйте этим шагам:‍
‍
1. Создайте новый репозиторий на GitHub:
   - Войдите в свой аккаунт на GitHub.
   - Нажмите на кнопку "New" (создать) в верхнем правом углу страницы.
   - Введите имя репозитория, выберите опцию создания README файла (если нужно), и нажмите "Create repository".‍
‍
2. Инициализируйте локальную папку как Git репозиторий:
   - Откройте командную строку или терминал в папке, которую вы хотите связать с репозиторием.
   - Используйте команду git init для инициализации локальной папки как Git репозитория.‍
‍
3. Свяжите локальный репозиторий с удаленным репозиторием на GitHub:
   - Скопируйте URL вашего удаленного репозитория на GitHub.
   - Используйте команду git remote add origin <URL> для привязки локального репозитория к удаленному на GitHub. Замените <URL> на URL вашего репозитория.‍
‍
4. Добавьте файлы, выполните коммит и отправьте их на GitHub:
   - Добавьте файлы в локальный репозиторий с помощью команды git add . (добавить все файлы) или git add <file> (добавить конкретный файл).
   - Выполните коммит с помощью команды git commit -m "Commit message" для сохранения изменений.
   - Отправьте изменения на удаленный репозиторий с помощью команды git push origin master. Это отправит изменения в ветку "master" на удаленном репозитории.‍
‍
Теперь ваш локальный репозиторий связан с удаленным на GitHub, и вы можете делать коммиты и отправлять изменения на сервер.‍
